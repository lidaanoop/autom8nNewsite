Autom8n Cluster has three components
1. Two way file synchronization using unison.
2. Two way database synchronization using MYSQL master master replication.
3. Configuration synchronization using csync 2
Using the above tools Autom8n creates replicas of the web application across multiple servers. You can then use the in built DNS load balancer or an external http load balancer to load balance connection among these servers. The servers act as active-active and connections can easily fail over to the available nodes when one of the replica is down.

Setting up a cluster is bit more involved than setting up a standalone server. There is also a steep learning curve. Due to the above two reasons we only provide fully managed services which includes installation/setup. The plans also come included with 24x7 monitoring of the cluster components and replication status.

1. Round Robin DNS load balancing - Autom8n sets up multiple A Records for the DNS resources in this topology. This is done on the existing bind/powerDNS DNS service. This is compactable with cPanel DNS clustering too.
2. Geo DNS Loadbalancing - Autom8n sets up specialized GDNSD software that does geographic loadbalancing and failover. This is superior to Round Robin DNS load balancing as it uses server side failover.

Yes, you can use an external load balancer but note that Autom8n will not configure the external load balancer and this must be done manually.
